# basic — Azure で手を動かしてネットワークを学ぶ

実際に Azure 上にリソースを構築し、通信を流し、設定を出し入れしながら、ネットワークの一般的な考え方と Azure での実現方法をあわせて身につけるための学習群です。

各ステップの具体的な内容は [`CLAUDE.md`](./CLAUDE.md) と各ステップの `README.md` を参照してください。ここでは **全体を通して何を、どのレベルまで学べるか** をまとめます。

## このステップ群で学べること

### ネットワークの一般概念
- アドレッシングとセグメント化（プライベート IP、サブネット分割、CIDR）
- 通信制御の基本軸 ―― **「許可／拒否」「経路」「名前解決」「外向き／内向き」** がそれぞれ独立したスイッチであること
- 最小権限の考え方（誰から・どこへの通信だけを開けるか）と、それを多層で組む設計
- L4（IP・ポート）と L7（HTTP・URL）の違い、負荷分散・リバースプロキシ・TLS 終端
- 踏み台・多段アクセス、egress の集約と検査、閉域構成といった実務的なパターン
- 疎通を「体感する」だけでなく「観測・記録する」診断の考え方

### Azure における通信・構成
- VNet / サブネット / NSG / VM といった基本リソースと、その組み合わせ方
- VNet ピアリング、ハブ&スポーク、UDR・NVA による経路制御
- NAT Gateway / Azure Firewall による外向き通信の制御
- Azure Bastion による踏み台のマネージド化
- Private DNS Zone・Private Endpoint / Private Link による名前解決と閉域接続
- Azure Load Balancer（L4）・Application Gateway（L7）による負荷分散
- Network Watcher による診断・フローログでの観測

### 到達できるレベル
- 基本リソースを **自分で組み合わせて、目的の通信構成を一から構築できる**
- 「通った／通らない」を確認するだけでなく、**設定を出し入れして因果（何が通信を成立・遮断させているか）を切り分けて説明できる**
- マネージドサービスと手組み構成を **対比して、責務・コスト・運用の違いを理解した上で選べる**
- 概念を Azure の具体リソースに対応づけて、**一般論と実装の両面から語れる**

### 学ばないこと（スコープ外）
範囲を「クラウド内のネットワークの基礎」に絞るため、以下は扱いません。必要になった時点で別途学ぶ前提です。
- **オンプレミス／ハイブリッド接続** ―― VPN Gateway、ExpressRoute、拠点間 VPN
- **大規模・広域設計** ―― Virtual WAN、マルチリージョン構成、可用性ゾーンをまたぐ冗長設計
- **動的ルーティング** ―― BGP など（経路は UDR の静的設定までを扱う）
- **DNS の内部実装** ―― 名前解決は「引ける／引けない」の動作確認まで。再帰解決やパブリック DNS 運用の深掘りはしない
- **セキュリティの作り込み** ―― WAF/IDS/IPS のチューニング、DDoS Protection、証明書・鍵の運用管理
- **コンテナ／PaaS 固有のネットワーク** ―― AKS の CNI、各 PaaS の専用ネットワーク機能（Private Endpoint で扱う範囲を除く）
- **運用・最適化** ―― 帯域／パフォーマンス設計、コスト最適化、本番運用の監視設計
- **IaC・自動化そのものの深掘り** ―― Bicep / just / Azure CLI は学習の道具として使うだけで、それ自体を体系立てて学ぶ対象にはしない

## 学び方の特徴
- 構築は Bicep、コマンドのまとめは just、Bicep でカバーしない部分は Azure CLI を利用
- 各ステップは前のステップを土台に積み上がり、新しい用語・概念は各ステップの `KNOWLEDGE.md` に、構成図は `MERMAID.md` に整理されている
- 共通する学習手法は **「設定を出し入れして、通信の変化から因果を確かめる」** こと

> 構築・実行はユーザー自身が行います（AI が Azure 上で実行することはありません）。
