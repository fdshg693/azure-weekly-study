# Step 1: Azure ネットワークの基礎と NSG によるアクセス制御

このステップでは、Azure 環境における最も基本的なネットワーク構成を構築し、ネットワークセキュリティグループ (NSG) のルールを変更することで、通信の許可・拒否がどのように機能するかを実際に確認します。

## 目的
* Azure の基本的なネットワークリソース（仮想ネットワーク、サブネット、NSG、パブリックIP、ネットワークインターフェース）の構成を学ぶ。
* NSG の受信セキュリティルールを変更し、アクセス制御（今回は ICMP / ping）が即座に反映されることを体験する。
* **[重要]** Azure において VM をデプロイするには VNet とサブネットが必須であること、また、今回のアクセス制御 (NSG) が **サブネットに対して適用されている** という基本アーキテクチャを理解する。

## 前提条件
この手順を実行するには、以下のツールがインストールされている必要があります。
* [Azure CLI](https://learn.microsoft.com/ja-jp/cli/azure/install-azure-cli) (事前に `az login` でログインを済ませておいてください)
* [Just](https://github.com/casey/just) (コマンドランナー)

## 構成されるリソース
`main.bicep` ファイルにより、以下のリソースがデプロイされます。
* **リソースグループ**: `rg-network-learn-minimal` (東日本リージョン)
* **仮想ネットワーク (VNet)**: `vnet-minimal` (アドレス空間: 10.0.0.0/16)
* **サブネット**: `subnet-default` (アドレス空間: 10.0.0.0/24)
  * Azure で VM をネットワークに参加させるための必須リソースです。
  * **今回はこのサブネットに対して NSG を関連付け、アクセス制御を行っています。**
* **ネットワークセキュリティグループ (NSG)**: `nsg-minimal`
  * サブネットに関連付けられ、初期設定で ICMP と SSH (TCP 22) の受信を許可しています。
* **パブリック IP アドレス**: `pip-vm-minimal` (VM に外部からアクセスするための IP)
* **ネットワークインターフェース (NIC)**: `nic-vm-minimal`
* **仮想マシン (VM)**: `vm-minimal` (Ubuntu 22.04 LTS, Standard_B1s)

## 実行手順

コマンドはすべてこの `step1` ディレクトリで実行してください。

### 1. リソースのデプロイ
以下のコマンドで、Azure 上にリソースを作成します。完了まで数分かかる場合があります。
```bash
just deploy
```

### 2. 疎通確認（Ping テスト）
デプロイが完了したら、VM のパブリック IP アドレスに対して ping を実行し、通信が成功することを確認します。
```bash
just test-ping
```
> 初期設定では NSG で ICMP (Ping) が許可されているため、応答が返ってきます。

### 3. 通信のブロック（Deny）をテスト
NSG のルールを更新して、ICMP 通信を拒否 (Deny) します。
```bash
just deny-ping
```
ルール更新後、再度 `just test-ping` を実行してください。今度は Ping の応答がなくなり、タイムアウトすること（通信がブロックされていること）が確認できます。

### 4. 通信の許可（Allow）をテスト
NSG のルールを再度更新して、ICMP 通信を許可 (Allow) に戻します。
```bash
just allow-ping
```
ルール更新後、再度 `just test-ping` を実行すると、再び Ping が通るようになります。

### 5. リソースの削除 (クリーンアップ)
テストが完了したら、無駄な課金を防ぐために以下のコマンドでリソースグループごと削除します。
```bash
just destroy
```
> ※削除処理はバックグラウンドで開始されます。完了まで数分かかります。
