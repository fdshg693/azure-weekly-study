# KNOWLEDGE.md (Step9)

このステップで新たに登場した用語や概念について解説します。

## Azure Load Balancer (L4 負荷分散)
Azure Load Balancer は、OSI 参照モデルの第4層（トランスポート層、TCP/UDP）で動作する負荷分散サービスです。
クライアントからの受信トラフィックを、送信元IP・送信元ポート・宛先IP・宛先ポート・プロトコルの「5タプル」に基づいて、複数のバックエンドリソースへ振り分けます。
Application Gateway（L7）のような URL パスやホスト名ベースの高度なルーティング機能はありませんが、低遅延で高スループットな分散処理が可能です。

## フロントエンド IP 構成 (Frontend IP Configuration)
ロードバランサーがクライアントからトラフィックを受け取るための「入り口」となる IP アドレスです。
インターネットからのアクセスを受け付ける場合は**パブリック IP**を割り当て、VNet 内部からのアクセスを受け付ける場合は**プライベート IP**（内部ロードバランサー）を割り当てます。

## バックエンドプール (Backend Pool)
ロードバランサーがトラフィックを振り分ける先の「受け皿」となるリソース（VM のネットワークインターフェイスや仮想マシンスケールセットなど）のグループです。
バックエンドプールに複数の VM を追加することで、システムの可用性と処理能力（スケール）を向上させることができます。

## ヘルスプローブ (Health Probe)
ロードバランサーがバックエンドプールの各インスタンスの「健康状態」を定期的に監視するための仕組みです。
指定したプロトコル（TCP, HTTP, HTTPS）とポートに対して定期的にリクエストを送信し、応答がない（連続して失敗する）インスタンスを「異常」と判断して、そのインスタンスへのトラフィックの振り分けを一時的に停止します。
正常な応答が再開されれば、自動的にトラフィックの振り分けを再開します。

## 負荷分散規則 (Load Balancing Rule)
フロントエンド IP で受け取ったトラフィックを、どのプロトコル・どのポート番号で、バックエンドプールのどのポート番号へ転送するかを定義するルールです。
この規則にフロントエンド IP、バックエンドプール、ヘルスプローブを紐付けることで、初めて負荷分散が機能します。
