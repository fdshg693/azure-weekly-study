# Step9: 受信トラフィックの負荷分散（L4）— Azure Load Balancer

このステップでは、**Azure Load Balancer** を用いて、複数のバックエンド VM に対して受信トラフィックを分散（負荷分散）する構成を学びます。これは「候補B」として計画されていたものです。

## 学ぶ概念
- **L4（TCP/UDP）の負荷分散**: クライアントからの接続を、IP アドレスやポート番号（L4 の情報）に基づいてバックエンドのリソースへ振り分けます。
- **バックエンドプール**: 負荷分散の対象となる VM（または VMSS）のグループです。
- **ヘルスプローブ**: ロードバランサーがバックエンド VM の「健康状態（サービスが正常に稼働しているか）」を定期的に確認する仕組みです。異常を検知した VM にはトラフィックを振り分けません。
- **高可用性とスケール**: 1つの公開エンドポイント（Public IP）の裏に複数の VM を並べることで、一部の VM がダウンしてもサービスを継続でき、アクセス数に応じて VM を増やすことでスケールアウトが可能になります。

## 構成
- **VNet**: `vnet-lb`
- **Subnet**: `subnet-lb`
- **NSG**: `nsg-lb` (HTTP ポート80、SSH ポート22 を許可)
- **Azure Load Balancer**: `lb-public` (Standard SKU)
  - フロントエンドIP: パブリックIP (`pip-lb`)
  - バックエンドプール: `BackendPool1` (2台の VM の NIC が所属)
  - ヘルスプローブ: TCP ポート80 (5秒間隔、2回の失敗で異常と判定)
  - 負荷分散規則: TCP ポート80 をバックエンドのポート80へ転送
- **VM**: `vm-backend-1`, `vm-backend-2` (Ubuntu 22.04)
  - Cloud-init (`customData`) を用いて、起動時に自動で Nginx をインストールし、Web ページにそれぞれのホスト名を出力するように設定しています。

## 手順

### 1. リソースのデプロイ

```bash
cd step9
just deploy
```
デプロイが完了すると、Load Balancer の Public IP が出力されます。
> **Note**: VM が起動した後、バックグラウンドで `apt-get install nginx` などの初期化処理が走るため、Web サーバーが応答を返すまでに3〜5分程度かかる場合があります。

### 2. 負荷分散の確認

デプロイ後、少し待ってから `test.sh` を実行します。

```bash
just test
```

次のように、2台の VM (`vm-backend-1`, `vm-backend-2`) から交互（またはランダム）に応答が返ってくることが確認できます。

```text
Hello from vm-backend-2
Hello from vm-backend-1
Hello from vm-backend-2
Hello from vm-backend-2
```
> Azure Load Balancer はデフォルトで「5タプルハッシュ（送信元IP、送信元ポート、宛先IP、宛先ポート、プロトコル）」を用いて振り分け先を決定します。そのため、同じ送信元からの連続したリクエストが同じバックエンドに偏ることもありますが、しばらくすると別の VM にも振り分けられるのが確認できます。

### 3. フェイルオーバー（異常時の動作）の確認

ロードバランサーの醍醐味である「1台がダウンしてもサービスが止まらない」ことを確認します。
別ターミナルを開き、1台の VM を意図的に停止（割り当て解除）します。

```bash
just stop-vm1
```

`just test` を実行したままのターミナルを観察すると、最初は片方の VM がダウンしたことによるタイムアウトが発生する可能性がありますが、ヘルスプローブが異常を検知した後は、生き残っている `vm-backend-2` だけに応答が寄り、エラーが起きなくなることが確認できます。

```text
Hello from vm-backend-2
Hello from vm-backend-2
Hello from vm-backend-2
```

## クリーンアップ

検証が終わったらリソースグループごと削除します。

```bash
just cleanup
```
