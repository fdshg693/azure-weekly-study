認証から認可へ ― App ロールで「何をしてよいか」を変える

同じユーザーでログインしても、割り当てられた App ロール（Tasks.Read / Tasks.Write）によって、呼べる API が変わります。 トークンに乗る scp（アプリの許可）と roles（ユーザーの役割）の違いに注目してください。

ロールの割り当ては task assign -- Tasks.Read / task assign -- Tasks.Write、解除は task unassign -- ...。 変更後はこの SPA が毎回トークンを強制リフレッシュするので、ボタンを押し直すと新しい roles が反映されます。
