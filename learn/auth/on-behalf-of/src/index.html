トークンを引き継ぐ ― On-Behalf-Of で多段 API にアイデンティティを伝播

SPA（あなた）→ 中間 API(A) → 下流 API(B) の多段呼び出し。A が受け取るトークンは aud=api://A なので、 そのまま B（aud=api://B）には使えません。A が On-Behalf-Of(OBO) 交換でトークンを作り替えて初めて、 あなたとして B まで到達できます。

chain-naive は生トークンをそのまま B へ転送 → B が aud 不一致で 401（失敗を見るボタン）。
chain-obo は A が OBO 交換してから B を呼ぶ → 200、B の応答にあなたの name が乗る（成功を見るボタン）。
chain-obo が 502 になるときは A→B の委任同意が未付与。task consent で付与、task revoke-consent で取り消し。
